Моделі (Models) використовуються для роботи з базою даних через Eloquent ORM. У проєкті основними є моделі Customer та Ticket.
Модель Customer відповідає за збереження даних клієнта (ім’я, email, телефон). Для створення або пошуку клієнта використовується метод firstOrCreate, який перевіряє чи існує клієнт з таким email, і якщо ні — створює нового.
Модель Ticket відповідає за збереження заявки. Метод create() використовується для створення нової заявки з вказаними полями: customer_id, subject, text та status. Через поле customer_id реалізується зв’язок між заявкою та клієнтом.
Також у моделі Ticket використовується метод addMedia(), який додає прикріплений файл до заявки за допомогою бібліотеки Spatie Media Library. Метод toMediaCollection('files') визначає колекцію, у якій зберігаються файли заявки.
Контролери (Controllers) відповідають за обробку HTTP-запитів. Контролер отримує дані з форми або API, передає їх для обробки у сервісний шар та повертає відповідь клієнту.
Form Request (Request) використовується для валідації вхідних даних. У класі StoreTicketRequest визначені правила перевірки для полів заявки (name, email, phone, subject, text). Це дозволяє винести логіку перевірки з контролера та зробити код більш структурованим.
API Resource (Response) використовується для формування відповіді API у форматі JSON. Клас TicketResource визначає, які поля заявки повертаються клієнту після створення заявки.
